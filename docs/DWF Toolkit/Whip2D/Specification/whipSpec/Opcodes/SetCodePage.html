Set Code Page

Purpose

    The Set Code Page function associates an operating system multi-byte character set to the World Wide Web URLs that follow.
Syntax
 
Opcode format Opcode Operand Format Comments
Extended ASCII (CodePage <ws><Ivalue>) Assigns (or removes) a code page.
    value   An integer that defines a valid code page for the operating system.
Details
    Text is stored in WHIP! data as Unicode. Thus, a code page is not necessary. URLs are also stored as Unicode, so it may appear that once again the code page is not necessary. The reason the code page is provided is based on the way internet browsers and internet servers interact.

    The strategy for placing a code page inside WHIP! data is based on the assumption that WHIP! data generated on one language system will be hosted on a server of the same langauge. In other words, WHIP! data generated on a Japanese system will probably be hosted on a internet server machine that is running Japanese. By having the code page information in the WHIP! data, viewers can use this information when converting URLs from Unicode to MCBS, if necessary.

Default
    By default the "Code Page" attribute is 1252 (English) which is equivalent to: 
      (Code Page 1252)
