Define Orientation - DEPRECATED

Purpose

Define Orientation represents the relationship between the paper and image orientation of the graphics block that the opcode represents. Currently supported formats are, Always_In_Sync, Always_Different and Decoupled. For example, if the orientation is formatted Always_In_Sync, then a portrait paper orientation will always force portrait graphics orientation and a Landscape paper orientation will always force landscape graphics orientation. Syntax
 
Opcode format Opcode Operand Format Comments
Extended ASCII (Orientation <ws><Torientation> -
Extended Binary 0x0145 <USorientation> -
    Orientation — This is one of the following ASCII strings in extended ASCII form:
      "Always_In_Sync"
      “Always_Different”
      “Decoupled”
       
      Orientation is one of the following enumerated values in extended binary form:
      0 = Always_In_Sync,
      1 = Always_Different
      2 = Decoupled
Details This opcode is always serialized as part of the BlockRef opcode and should always be uncompressed. All of the fields are serialized to the maximum field-length bytes. Default By default,  Orientation is  Decoupled.
