Set Line Pattern

Purpose

    The Set Line Pattern function specifies the repeating pattern with which lines are rendered.
Syntax
 
Opcode format Opcode Operand Format Comments
ASCII Operand (LinePattern <ws><TPatName>[<ws>])  Defines how subsequent lines are rendered.
Binary Operand 0xCC <BPatNum>[<USEpatnum>]  Corresponds to the pattern of subsequent lines.
ASCII Operand (DashPattern <ws><IDashpatnum> <ws><[<IPixels_on>,<IPixels_off>]+[<ws>])  Application defines the pattern. 
    PatName  An ASCII text string that describes a pattern with which to render subsequent lines

    PatNum  A number that corresponds to a pattern with which to render subsequent lines

    Epatnum  In the binary form, if the pattern number is greater than 255, then PatNum is stored as zero, and a two byte Epatnum follows indicating layer numbers from the range 256 to 65791

    Dashpatnum  A number that corresponds to a pattern defined by the application

    • Pixels_on Represent pixels that are lighted
    • Pixels_off Represent pixels that are not lighted
Details
    The Set Line Pattern opcode specifies a pattern of dashes and dots with which subsequent lines are rendered. The attribute applies to lines, polylines, circular arcs, and elliptical arcs. Applications can specify patterns in WHIP! data via two ways:
    • Line Pattern Allows the application to select one of 31 predefined patterns that are associated with WHIP! data. For ASCII WHIP! data, applications use Line Pattern by specifying the name of the pattern. For binary WHIP! data, a number corresponding to a particular pattern is specified.
    • Dash Pattern Allows the application to define its own pattern rather than use the predefined patterns. A desired pattern is specified by indicating how many pixels are on and off.
    Note: Line patterns have been extended from values of 18 through 31 to include ISO linetypes. This should reduce tessellation of  ISO linetypes that are tessellated via the ePlot driver through Heidi. Notes

    Because the way that line patterns are rendered is highly dependent upon the WHIP! data reading application, line patterns are meant to be used only as a visual cue. Normally, a WHIP! data reading application renders line patterns in screen space, such that the size of the pattern does not change as a user zooms-in on the drawing. If exact line patterns are required that synchronize with other geometric primitives in the drawing, then the WHIP! data writing application should explicitly draw the individual dashes and dots required as a series of  Draw Line calls.
