Merge Control

Purpose

    Merge Control affects the behavior of overlapping objects. If it is set, the Merge Control opcode signifies to the viewer that, a raster op has to be performed on drawables in order to merge objects, whenever the drawables overlap.
Syntax
 
Opcode format Opcode Operand Format Comments
Extended ASCII (LinesOverwrite <ws>'opaque'|'merge'|'transparent -

Details

    Currently Merge Control is supported as an enumerated value. Three states are supported: "lines overwrite", "lines merge" and "transparent."

    Merge control is set only in the WHIP! data. If set, Merge Control state applies to the drawable that follows it. In otherwords, Merge Control is supported as a rendition attribute for drawables.

    Only Extended  ASCII is supported,  there is no binary or extended binary support. Use Cases

    Draw a few black lines. Then draw a light gray solid over them. If the state is "lines overwrite", then the light gray solid will obscure the black lines in a gray-scale plot. If  the state is "lines merge," the black lines will be visible.
