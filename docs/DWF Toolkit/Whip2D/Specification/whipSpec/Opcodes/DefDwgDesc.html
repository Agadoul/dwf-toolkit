Define Drawing Description - DEPRECATED

Purpose

    The Define Drawing Description function specifies a description of what the WHIP! data represents.
Syntax
 
Opcode format Opcode Operand Format Comments
Extended ASCII (Description <ws><TDesc>) Explains the contents of the WHIP! data.
    Desc String describing what is represented by the WHIP! data
Details
    Some WHIP! data reading applications may need to "know" what the image in the WHIP! data represents. The WHIP! data generating application can indicate this with the description opcode:
      (Description 'Blueprints for Pam’s new office')
Notes
    WHIP! data reading applications should not expect to be able to parse the Desc argument, though it could be shown to a user.

    WHIP! data should contain a maximum of one occurrence of this opcode, which by convention should be nested inside a Define Drawing Information Block opcode.

Default
    By default, no drawing subject is defined, which is equivalent to
      (Description unknown)
