Define Drawing Modification Time - DEPRECATED

Purpose

    The Define Drawing Modification Time function provides information to WHIP! data reading applications about when the WHIP! data was last modified.
Syntax
 
Opcode format Opcode Operand Format Comments
Extended ASCII (Modified <ws><ITime><ws><TDesc><ws><TGUID>) Defines when file was last changed
    Time   The number of seconds elapsed since midnight (00:00:00), January 1, 1970, coordinated universal time (GMT)

    Desc   Any humanly readable description of the time when the file was last modified

    GUID   A globally unique identifier (GUID) represented by an alpha-numeric description generated to capture the modification time of the WHIP! data. When WHIP! data is initially created, the creation and modification GUIDs match.

Details
    Some WHIP! data reading applications may need to "know" when the WHIP! data was last changed. The WHIP! data generating application can indicate this with the Modified opcode:
      (Created 820497600 'January 1, 1996' C0176e40-dc10-11d0-bb09)
Notes
    WHIP! data reading applications should not expect to be able to parse the Desc argument, though it could be shown to a user.

    WHIP! data should contain at most one occurrence of the modification time opcode, which by convention should be nested inside a Define Drawing Information Block opcode.

Default
    By default no drawing modification time is defined, which is equivalent to
      (Modified 0 unknown unknown)
