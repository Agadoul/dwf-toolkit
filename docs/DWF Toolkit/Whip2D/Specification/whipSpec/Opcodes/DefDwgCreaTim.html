Define Drawing Creation Time - DEPRECATED

Purpose

The Define Drawing Creation Time function provides information to WHIP! data reading applications about when the WHIP! data was generated. Syntax
 
Opcode format Opcode Operand Format Comments
Extended ASCII (Created <ws><ITime><ws><TDesc><ws><TGUID>) Defines when file was generated
Time The number of seconds elapsed since midnight (00:00:00), January 1, 1970, coordinated universal time (GMT)

Desc Any humanly readable description of the time when the file was generated

GUID A globally unique identifier (GUID) represented by an alpha-numeric description generated to capture the creation time of the WHIP! data

Details Some WHIP! data reading applications may need to "know" when the WHIP! data was created. The WHIP! data generating application can indicate this need with the Created opcode: (Created 820497600 'January 1, 1996' C0176e40-dc10-11d0-bb09) Notes WHIP! data reading applications should not expect to be able to parse the Desc argument, though it can be shown to a user.

WHIP! data should contain a maximum of one occurrence of the creation time opcode, which by convention should be nested inside a Define Drawing Information Block opcode.

 Default By default no drawing creation time is defined, which is equivalent to (Created 0 unknown unknown)
